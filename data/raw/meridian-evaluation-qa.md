## Format
Each entry contains:
- **Question** — the query sent to the RAG pipeline
- **Expected Answer** — the ground truth answer derived from the handbook
- **Source Section** — the handbook section the answer comes from
- **Difficulty** — Easy / Medium / Hard

## Usage Notes

- Run each question through the pipeline **without RAG** first and record the response. The model should either answer incorrectly or state it does not have this information.
- Run each question **with RAG** and record the response. Compare against the expected answer.
- Use the before/after comparison in your README and Gradio demo to clearly illustrate the value of the pipeline.
- For `eval.py`, consider scoring each answer on: **Correctness** (does it match the expected answer), **Faithfulness** (is it grounded in the retrieved chunk), and **Relevance** (does it address the question).
